# Agile Engineering - Go_TOFU

This docs site is the single source of truth for planning, research, and execution.

## Quick Start

1. Read `Roadmap` for goals and current sprint focus.
2. Update `Task Board` for execution status.
3. Add interview insights in `Interview Pipeline`.
4. Track market movements in `Competitor Tracker`.
5. Log decisions in `Meeting Notes`.

## Current Problem Statement

- User interviews are slow, expensive, and hard to scale.
- Teams often cannot access enough target users early.
- Existing data (reviews, surveys, transcripts) is underused.
- Expert feedback loops are resource-intensive.

## Strategy Snapshot

- Build synthetic personas from public + proprietary data.
- Simulate interviews quickly for faster product decisions.
- Turn broad qualitative data into structured insights.

## Team Workflow

- Weekly planning in `Roadmap`
- Daily updates in `Task Board`
- Ongoing evidence capture in `Interview Pipeline`
- Weekly synthesis in `Meeting Notes`

## Documentation Rules

- Keep pages short and scannable.
- Add dates using `YYYY-MM-DD`.
- Prefer bullets over long paragraphs.
- Update status fields every sprint.
